Team Maker - Website

A website to create balanced League of Legends teams.

A Short Description

Context

Creating a website so the League of Legends community staff can quickly and easily balance teams for events

Time

1 week

Tech Stacks Used

Vue, Javascript, HTML/CSS, Google Sheets API

An In-Depth Look

After being promoted to Head of Events of Michigan Esports, my position as League of Legends community director was passeed down to other students who now work under me on the event commitee. I wanted to turn my team balancing script into a website that my League of Legends community directors can use to create balanced League of Legends teams.

I recently organized a gaming event that over 125 people came to. Over 30 people game for League of Legends. For the next event I'll be holding, I'll be telling my staff to use this website to save a lot of time.

The Planning

I already wrote the algorithm in python. Since I'm currently taking 493 User Interface Development and learned about Vue.js in lecture, I decided it would be good practice to use Vue and Javascript for this project.

Here's my process.

  1. I first drew a draft interface on Concepts.
  2. I started doing a bunch of research of what I could do in terms of accessing the Google Sheets for information.
    • My previous script used a service account to access a certain spreadsheet. If I used a service account again, I would have to somehow need to have the user share their sheet with the service account.
    • I could have the website share the form that the user should use to collect data. There would be two buttons on the website; one to run the script on the information on the sheet, one to clear the forms/the spreadsheet.
  3. The first option seemed like a hassle to continue sharing the sheet with a suspicious service account name. The second option seemed like it could work, but Google Sheets doesn't have an option to clear forms.
  4. Luckily, creating an API key to view public spreadsheets seemed to be the best solution. So, that's what I went with.
  5. I tested the API calling by typing in a sheet_id using Vue and Axios to fetch.
  6. I recoded my Python script into Javascript
  7. I use Vue to dynamically change the page and display the matches.

Having the script in Python origianlly made the algorithm part of this project much easier to code. I was able to reuse more code this time around by creating more generic functions.

How It Works

Check out my Team Maker Script page to see how the algorithm works. Below is how a user would experience this website:

  1. A community director makes a copy of the linked form on the website and send it out to all the players that want to play.
  2. Once all the users have filled it out, the community director publishes the sheet to the web.
  3. The community director takes the sheet_id and inputs it into the website text input.
  4. They press "Generate" to generate teams
  5. The teams are all outputted onto the website dynamically.
